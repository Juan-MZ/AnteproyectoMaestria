\section{Cronograma}

El proyecto se plantea para ejecutarse en \textbf{cinco meses}, con una dedicación promedio de \textbf{9 a 10 horas por semana}, para un total de \textbf{192 horas}. El cronograma adopta una lógica iterativa: cada mes cierra con una revisión de avance, ajuste del backlog técnico y validación de entregables con el director.

Para mantener el cronograma bajo control, al cierre de cada mes se realizará una revisión de entre 1 y 2 horas con el director. Esa revisión permitirá contrastar horas planeadas frente a horas reales, depurar el alcance si aparece sobrecarga y priorizar el siguiente ciclo de trabajo sin comprometer los objetivos centrales del proyecto.

\noindent
La distribución de horas por tipo de trabajo quedará así: \textbf{32 h} para revisión y definición de reglas, \textbf{80 h} para diseño e implementación del prototipo, \textbf{42 h} para evaluación controlada y ajustes, y \textbf{38 h} para documentación, reuniones y cierre. Esta distribución concentra el mayor esfuerzo en la construcción del artefacto, pero conserva tiempo suficiente para validar resultados y documentar el proceso de manera académicamente defendible.
